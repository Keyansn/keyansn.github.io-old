IoTea

Internet of Things smart tea brewer

Overview

For Third year Embedded Systems course, we were assigned to collaborate in teams of four to design an Internet of Things device with a given random sensor and an Adafruit Huzzah Feather board. We were given a TSL2561 luminosity sensor which outputted a lux value via an I2C connection. Given that tea changes opacity during the steeping process, which is measurable with said sensor, we decided to develop a prototype of a smart tea brewer which could detect when the tea had reached the predefined strength. A connection to an MQTT server was established and from which the prototype was able to communicate with a MySQL server.

I2C: This was the first time encountering the I2C protocol so a solid understanding had to be achieved. As we were developing using MicroPython, there was additional work needed as we had to create the driver ourselves.

MQTT: Part of the specification was that the prototype created must be able to communicate with a pre setup MQTT server. This was done so that the data for the different strengths of teas could be sent to the device as well as from the device back to the server to message when done.

MySQL: A MySQL database was set up for storing this tea making data. Each tea requires very specific opacity values and temperature to ensure accurate reproduction every time.
